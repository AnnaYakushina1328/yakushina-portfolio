# Личный сайт с образовательными и проектными материалами

Статический сайт для публикации на GitHub Pages или другом бесплатном хостинге. Сайт оформлен как личная страница с открытыми материалами, проектами и образовательной активностью, без привязки к конкретной анкете или резюме.

## Структура

- `index.html` — главная страница.
- `vkr.html` — отдельная страница дипломного проекта.
- `activities.html` — страница с проектной активностью, хакатонами и наставничеством.
- `styles.css` — оформление сайта.
- `materials/` — папка для открытых материалов, заметок, конспектов и ссылок.

## Что заменить перед публикацией

1. В блоке «Контакты» заменить `example@example.com` на актуальную почту.
2. При необходимости добавить ссылку на GitHub, облачную папку или профессиональный профиль.
3. В разделе `materials/` заменить черновые markdown-файлы на реальные материалы.
4. Добавить скриншоты проекта, презентацию или ссылку на репозиторий, если они готовы к публикации.
5. Для хакатонов можно добавить ссылки на страницы мероприятий или подтверждающие материалы, если их можно публиковать.

## Важно перед публикацией документов

Не стоит выкладывать в открытый доступ сканы диплома, приложения к диплому, паспорта, согласия и другие документы с датой рождения, регистрационными номерами, подписями или печатями. Для сайта лучше использовать обезличенные версии: аннотацию, описание проекта, презентацию без лишних персональных данных или ссылку на репозиторий.

Также лучше не публиковать персональные данные студентов и участников команд без отдельного согласия. На сайте уже использованы нейтральные формулировки без перечисления обучающихся.

## Публикация на GitHub Pages

1. Создать публичный репозиторий, например `yakushina-materials`.
2. Загрузить в него файлы сайта.
3. Открыть `Settings` → `Pages`.
4. Включить публикацию из ветки `main`, папка `/root`.
5. После публикации сайт будет доступен по адресу вида: `https://USERNAME.github.io/yakushina-materials/`.
